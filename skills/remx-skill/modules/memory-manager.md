# MemoryManager

**核心决策引擎。** 负责语义分析、自主决策，以及协调各模块完成记忆操作。

Skill 层不等待用户提醒——MemoryManager 分析当前上下文后主动决定是否召回、创建或更新记忆。

---

## 核心职责

1. **上下文分析**：判断当前对话是否涉及项目相关讨论
2. **记忆价值判断**：判断是否包含值得创建或更新的内容
3. **决策执行**：调用 ContextAssembler / MemoryFileManager / DecayWatcher 执行操作
4. **摘要生成**：在 Agent 回答末尾追加记忆操作摘要

---

## 决策流程

```
每次 Agent 生成回答前：
┌─────────────────────────────────────┐
│ 1. 上下文分析                        │
│   → 讨论是否涉及项目内容？            │
│     （架构/方案/决策/术语/文件/模块）  │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│ 2. 记忆召回（RECALL）               │
│   → ContextAssembler.assemble()     │
│   → 注入相关记忆到 Agent prompt      │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│ 3. 回答生成                          │
│   → Agent 收到上下文，自然引用记忆    │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│ 4. 记忆价值分析                      │
│   → 输出了新决策/结论/方案？→ CREATE │
│   → 记忆内容与事实不符？→ UPDATE     │
│   → 无值得记录的内容？→ NONE        │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│ 5. 执行 + 摘要输出                   │
│   → MemoryFileManager.write/update   │
│   → 答案末尾追加 📚🆕🔄🔗 摘要       │
└─────────────────────────────────────┘
```

## 决策规则

### RECALL — 何时召回记忆

| 触发条件 | 说明 |
|---------|------|
| 讨论涉及项目架构/方案 | 用户问"模块 X 是怎么实现的" |
| 讨论涉及历史决策 | 用户问"之前为什么选了方案 Y" |
| 讨论涉及已知问题 | 用户提到某个 bug 或 issue |
| Agent 需要引用项目背景 | 回答需要提供上下文信息 |

**执行方式：** 调用 `ContextAssembler.assemble()`，将结果注入 Agent prompt。

### CREATE — 何时创建记忆

| 触发条件 | 说明 |
|---------|------|
| 输出了新的技术决策 | "我们决定用方案 A 而不是方案 B" |
| 输出了结论或答案 | 解决了某个技术问题并给出了方案 |
| 发现并记录了新问题 | "这是一个 bug，原因 是..." |
| 提出了新的待办 | "下一步应该做 X" |
| 写入了新的知识 | 输出了技术原理、设计思路 |

**创建优先级：**
- `demand` — 技术决策、设计方案
- `issue` — bug、风险、问题
- `knowledge` — 技术原理、架构思路
- `tmp` — 临时想法、会议记录
- `principle` — 规范、原则

**执行方式：** 调用 `MemoryFileManager.write()`，在 Agent 回答末尾追加 `🆕 新建: <path>`。

### UPDATE — 何时更新记忆

| 触发条件 | 说明 |
|---------|------|
| 发现现有记忆与事实不符 | "之前记录的方案 A 实际上用的是方案 B" |
| 决策变更需要更新记录 | "我们改用方案 C 了" |
| 发现记忆内容过时 | 之前的方案已被新方案替代 |
| 补充信息到已有记忆 | 在已有记忆上追加新的知识点 |

**执行方式：** 调用 `MemoryFileManager.update()`，在 Agent 回答末尾追加 `🔄 更新: <path>`。

### NONE — 无操作

以下情况不触发记忆操作：
- 用户闲聊（非项目相关）
- 用户问的是当前对话中新产生的临时信息
- Agent 只是在解释常识性知识

## 输出摘要格式

当有记忆操作时，在 Agent 回答**末尾**追加一行摘要：

```
---
📚 召回: demands/auth-module.md, issues/bug-token.md
🆕 新建: demands/session-token.md
🔄 更新: demands/auth-module.md
🔗 拓扑: demands/auth-module.md → issues/bug-token.md (因果关系)
```

**生成规则：**
- `📚 召回`：本次召回的记忆路径列表（去重）
- `🆕 新建`：本次新建的记忆路径
- `🔄 更新`：本次更新的记忆路径
- `🔗 拓扑`：本次新建的拓扑关系

**无操作时：** 不显示摘要行。

## 主动触发场景

以下场景 Agent **无需用户提醒**，自动执行记忆操作：

| 场景 | 自动操作 |
|------|---------|
| 会话开始时 | `DecayWatcher.check()` → 提醒快到期记忆 |
| 讨论项目架构/方案时 | `ContextAssembler.assemble()` → 自动召回 → 自然引用 |
| 输出了技术决策时 | `MemoryFileManager.write()` → 创建 demand 记忆 |
| 发现了 bug/问题时 | `MemoryFileManager.write()` → 创建 issue 记忆 |
| 记忆内容与事实不符时 | `MemoryFileManager.update()` → 更新记忆 |
| 引用了某条记忆时 | `remx relate insert` → 建立拓扑相关性 |
| `remx index` 完成后 | `DecayWatcher.check()` → 检查衰减 |

---

## 拓扑关系建立规则

| 条件 | 自动建立拓扑 |
|------|------------|
| 创建 demand 并解决了 issue | demand → issue（因果关系）|
| 引用了相关记忆 | 当前讨论主题 → 被引用记忆（相关性）|
| 方案 A 否决了方案 B | 方案 A → 方案 B（对立性）|
| 模块 A 依赖模块 B | 模块 A → 模块 B（依赖性）|
| 子模块是整体的一部分 | 子模块 → 整体（组成性）|

---

## 与其他模块的协作

```
MemoryManager（决策中心）
    ├── ContextAssembler   — 召回记忆
    ├── MemoryFileManager  — 写/更新/删除记忆文件
    ├── DecayWatcher      — 衰减检查
    └── remx CLI          — index/relate/gc 命令执行
```

---

## 实现注意事项

1. **幂等性**：相同内容重复创建时，不重复创建（front-matter + content_hash 判断）
2. **异步执行**：记忆写入在 Agent 回答之后，不阻塞回答生成
3. **错误处理**：写入失败时仅警告，不影响回答
4. **摘要合并**：同一次回答中多个操作时，合并为一个摘要行
